Technische Maßnahmen (TOM)

4.1 Verschlüsselung

  • Auf der Kamera: AES-256-Verschlüsselung aller Aufnahmen während der Aufzeichnung
  • Übertragung: Ende-zu-Ende-Verschlüsselung Hub → Server (TLS 1.3 + AES-256)
  • Speicherung: Verschlüsselung at-rest auf Server (AES-256)
  • Export: .mex-Containerformat, nur mit NetCo MediaPlayer abspielbar, kopiergeschützt

4.2 Zugriffskontrolle

  • Kamera: NFC/RFID-Karte oder PIN + UserID (kein Zugriff auf Rohdateien am Gerät)
  • AdminClient: Rollenbasiert (Admin, Supervisor, Viewer), 2FA empfohlen
  • MediaClient: Eingeschränkte Rechte, kein Export ohne Admin-Freigabe
  • Server: Netzwerksegmentierung, Firewall, nur LAN-Zugriff für SmartHubs

4.3 Integritätsschutz

  • Digitale Wasserzeichen mit Seriennummer, UserID, GPS-Koordinaten, Zeitstempel
  • MKV-Format mit eingebettetem Audio (Vollständigkeit)
  • Hash-Wert pro Aufnahme (Manipulationserkennung)
  • Audit-Trail: Jede Aktion (Zugriff, Export, Löschung) wird protokolliert

4.4 Datenlöschung

  • Automatische Löschung auf Kamera nach erfolgreichem Upload
  • Konfigurierbare Löschfristen im AdminClient ([72 Stunden] Standard)
  • Revisionssichere Löschprotokolle (wer hat was wann gelöscht)
  • Keine manuelle Löschung einzelner Aufnahmen (nur Admin mit Begründung)

Organisatorische Maßnahmen

  • IT-Sicherheitsbeauftragter wird in Konzept eingebunden
  • Betriebsvereinbarung regelt organisatorischen Rahmen
  • Schulungspflicht für alle Nutzer und Administratoren
  • Vierteljährlicher Sicherheitsaudit (Zugriffsrechte, Löschprotokolle, Firmware-Stand)
  • Incident-Response-Plan für Datenverlust/-diebstahl
  • Regelmäßige Firmware-Updates (NetCo stellt bereit)
  • Notfallplan: Bei Serverausfall → Kameras speichern lokal bis Reconnect
